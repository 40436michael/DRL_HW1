# 對話記錄 (Conversation Log)

這份文件記錄了在這個專案中整合您提供的程式碼以及作業需求的歷史開發軌跡。

## 📅 開發紀要與重點

### 🟢 階段一：實作基礎核心與策略評估 (HW1-1 & HW1-2)
- **初始狀態**: 完成了網格的基本生成與值迭代（Value Iteration）邏輯。
- **改進項目**: 將原先「等機率選擇全部四個方向」的隨機策略，修正為每個格子**挑選一個確定的隨機箭頭**。
- **目的**: 嚴格對齊評分標準 HW1-2 的「每個單元格的隨機生成行動（上下左右箭頭）作為策略」核心要求。

### 🟢 階段二：使用者自帶「質感升級」UI 模版
- **使用者要求**: 載入玻璃擬態卡片（Glassmorphism）、星空色漸層、且包含尋路動態回饋及 FontAwesome 圖示的全新專案架構。並請求預設盤面為：**大小 5x5、起點 (0,0)、終點 (4,4)，障礙物設於 (1,1)、(2,2)、(3,3)**。
- **實作結果**:
  - 全面覆蓋原有的基礎樣式。
  - 將專案拆分為 `templates/index.html`，以及靜態資源 `/static/js/script.js` 和 `/static/css/style.css` 乾淨整潔的架構。
  - 第一時間套用了使用者的預設盤面配置，點擊按鈕後展示精美的動畫漸變與路徑提示。

### 🟢 階段三：融合優美設計與作業絕對限制 (HW1 Strict Integration)
- **功能挑戰**: 上方的質感升級版 UI 將變數綁死在了 `5x5` 的硬編碼環境，這將遺失原先獲得 HW1 滿分的兩大要點（可調動態的格數 5-9 以及 $n-2$ 隨機行動評估）。
- **實作結果**:
  - **動態延伸性 ($n \times n$)**: 幫新的介面重新加入了拉桿滑桿 `$n$` (5~9)，且網頁會自動隨著滑桿調整 `grid-template-columns` 等 CSS 排版避免跑版。
  - **動態障礙物控制 ($n-2$)**: 當拉桿變更時，不僅清除越界的方塊，還強制綁定畫面的計數器以防使用者設置超過 `$n-2$` 的限制。
  - **HW1-2 隨機行動顯示整合**: 重寫後端，將對隨機行動做單純 Policy Evaluation 驗證的 `/evaluate_random_policy` 獨立為一個 Endpoint。
- **回饋與修正**: 修復未使用檔案造成的雜訊，清理了先前試錯的 `/css` 等多餘目錄。

### 🟢 階段四：專注於配色細節修正
- **使用者要求**: 將指定的 Start 保留為綠色，End 改為紅色，Block 改為灰色。其餘無相關的玻璃化背景格位改回**原始的純白色**。
- **實作結果**:
  - 調整 `style.css` 中的 `var(--end-color)` 與 `var(--block-color)` 為正宗的紅 `#ef4444` 及灰色 `#64748b` 以遵守要求。
  - 獨立抽離空網格方塊，將它的背景色調為絕對 `#ffffff` 白，並搭配深色色碼 `#1e293b` 來突顯白色底下的文字，去除了文字陰影避免白色上糊化。
